### ERROR 1 — Figma Asset Imports Not Resolvable

**Severity:** CRITICAL — Build fails
**Affected Files:** All major page and component files

**What's wrong:**
The project uses a non-standard import scheme (`../../assets/[hash].png`) in 80+ places throughout the codebase. These are generated by Figma Make and require a special Vite plugin to resolve. No such plugin or resolver is configured in `vite.config.ts`, so the build immediately fails when Rollup tries to bundle these imports.

**Example import (found in multiple files):**

```ts
import assetSomeName from "../../assets/abc123def456.png";
```

**Error you would see:**

```
[vite]: Rollup failed to resolve import "../../assets/..."
```

**Affected files include:**

- `src/main.tsx`
- `src/app/pages/Home.tsx`
- `src/app/pages/AboutUs.tsx`
- `src/app/pages/Members.tsx` (25+ imports)
- `src/app/pages/Sponsor.tsx`
- `src/app/pages/RoboSub2026.tsx`
- `src/app/components/Navbar.tsx`
- `src/app/components/Footer.tsx`
- All WFSA pages and Team sub-pages

**How to fix (choose one):**

**Option A — Replace Figma imports with real asset paths (Recommended for deployment)**
Extract the actual image files from Figma and place them in the `public/` or `src/assets/` folder, then update each import to a regular path:

```ts
// Before
import heroImage from "../../assets/abc123def456.png";

// After
import heroImage from "../assets/hero-image.png";
// or for public folder: const heroImage = "/assets/hero-image.png";
```

**Option B — Add a custom Vite resolver plugin**
If images are hosted somewhere (e.g., Figma CDN or your own server), add a Vite plugin in `vite.config.ts` to resolve `../../assets/` imports to actual URLs:

```ts
// vite.config.ts
plugins: [
  react(),
  {
    name: "figma-asset-resolver",
    resolveId(id) {
      if (id.startsWith("../../assets/")) {
        return id; // mark as virtual
      }
    },
    load(id) {
      if (id.startsWith("../../assets/")) {
        const hash = id.replace("../../assets/", "");
        // Return the URL where the asset lives
        return `export default "https://your-cdn.com/assets/${hash}";`;
      }
    },
  },
];
```

### ERROR 2 — React Router Import Package Mismatch

**Severity:** CRITICAL — Runtime crash
**Affected Files:** 10+ files

**What's wrong:**
All routing components (`BrowserRouter`, `Routes`, `Route`, `Link`, `useLocation`) are imported from `"react-router"`, but in React Router v7 (which is what `package.json` specifies: `"react-router": "^7.13.0"`), the browser-specific components live in `"react-router-dom"`, not `"react-router"`. Importing from the wrong package causes a runtime crash.

**Affected files and imports:**

- `src/app/App.tsx` — `BrowserRouter`, `Routes`, `Route`
- `src/app/components/Navbar.tsx` — `Link`
- `src/app/components/Footer.tsx` — `Link`
- `src/app/components/ScrollToTop.tsx` — `useLocation`
- `src/app/pages/RoboSub2026.tsx` — `Link`
- `src/app/pages/WFSA2023.tsx` — `Link`
- `src/app/pages/WFSA2024.tsx` — `Link`
- `src/app/pages/WFSA2025.tsx` — `Link`
- `src/app/pages/WFSA2023Team.tsx` — `Link`
- `src/app/pages/WFSA2024Team.tsx` — `Link`
- `src/app/pages/WFSA2025Team.tsx` — `Link`

**How to fix:**

**Step 1 — Install `react-router-dom`:**

```bash
npm install react-router-dom
```

**Step 2 — Update all imports in the above files:**

```ts
// Before
import { BrowserRouter, Routes, Route } from "react-router";
import { Link } from "react-router";
import { useLocation } from "react-router";

// After
import { BrowserRouter, Routes, Route } from "react-router-dom";
import { Link } from "react-router-dom";
import { useLocation } from "react-router-dom";
```

### ERROR 4 — React Listed as Peer Dependency Instead of Dependency

**Severity:** HIGH — Fragile build setup
**File:** `package.json` — Lines 73–83

**What's wrong:**
`react` and `react-dom` are listed under `peerDependencies` (with `optional: true`) instead of `dependencies`. This means they will not be reliably installed on fresh deploys (e.g., on Vercel, Netlify, or any CI environment), which will cause the build to fail with "Cannot find module 'react'" errors.

**Current state in `package.json`:**

```json
"peerDependencies": {
  "react": "18.3.1",
  "react-dom": "18.3.1"
},
"peerDependenciesMeta": {
  "react": { "optional": true },
  "react-dom": { "optional": true }
}
```

**How to fix:**
Move React to `dependencies`:

```json
"dependencies": {
  "react": "18.3.1",
  "react-dom": "18.3.1",
  ...other existing dependencies...
}
```

Then remove the `peerDependencies` and `peerDependenciesMeta` entries for React.
